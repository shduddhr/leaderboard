# D&A ML Session Leaderboard Server 🏆

D&A 머신러닝 스터디 세션을 위해 구축된 **자체 캐글형(Kaggle-like) 리더보드 플랫폼**입니다.
이 플랫폼은 고객 구독 해지(Churn) 여부를 예측하는 분류 문제를 해결하며, 7일 동안 참가자들이 결과를 제출해 순위를 나눌 수 있게 지원합니다.

---

## 1. 프로젝트 주요 특징

- **현실적인 합성 데이터 생성**:
  - `Age`, `Monthly_Bill`, `Support_Calls` 등 변수 간 복잡한 비선형 관계 및 노이즈가 포함된 10만 행 데이터 셋을 자체 생성하여 저작권 걱정이 없습니다.
- **실시간 Public 리더보드**:
  - 전체 테스트 셋의 **30% (Public)**로 실시간 스코어를 매겨 공개하며, 나머지 **70% (Private)**의 최종 점수는 대회 종료 후 한 번에 공개하여 치팅(오버피팅)을 방지합니다.
- **간이 로그인 기능**:
  - 회원가입 프로세스를 제거하여 팀이 첫 접속 시 **팀 이름**과 **비밀번호**를 지정해 즉시 참여할 수 있습니다.
- **제출 횟수 제한**:
  - 각 팀당 하루 제출 횟수를 기본 **5회**로 제한하여 오버피팅을 제한합니다. (제한은 `main.py`의 `MAX_DAILY_SUBMISSIONS` 변수로 손쉽게 조절할 수 있습니다.)

---

## 2. 로컬 실행 방법 (Local Run)

로컬에서 먼저 서버를 띄워 테스트할 수 있습니다.

### 의존성 패키지 설치
```bash
pip install -r requirements.txt
```

### 1단계: 데이터셋 생성
서버 구동 전에 반드시 가상 데이터를 먼저 생성해야 합니다.
```bash
python generate_data.py
```
- `data/` 디렉토리가 생성되며 다음 파일들이 생성됩니다:
  - `train.csv` (70,000행): 참가자에게 배포할 학습용 데이터
  - `test.csv` (30,000행): 참가자에게 배포할 예측 타겟 제외 데이터
  - `solution.csv` (30,000행): 채점용 정답 파일 (Public/Private 구분값 포함)

### 2단계: 서버 구동
```bash
uvicorn main:app --reload
```
서버 실행 후 브라우저에서 `http://127.0.0.1:8000`에 접속하여 화려한 Glassmorphism UI를 확인하세요.

## 3. 무료 클라우드 서비스(Koyeb / Render) 배포 가이드 🌐

서버가 꺼지지 않고 24시간 가동되도록 7일 동안 무료 클라우드에 배포하여 운영할 수 있습니다. **Koyeb**을 사용하는 방법을 적극 권장합니다. (Render 대비 빌드 속도가 매우 빠르고 무료 도메인이 기본 제공됩니다.)

### Koyeb으로 배포하기 (가장 추천)
1. **GitHub 리포지토리 생성 및 업로드**:
   - GitHub에 새로운 비공개(Private) 또는 공개(Public) 리포지토리를 만들고, `leaderboard/` 폴더 안의 모든 파일(Dockerfile, requirements.txt, main.py, database.py, templates/ 등)을 푸시합니다.
   - 단, `data/` 폴더와 `leaderboard.db`는 푸시하지 않아도 됩니다. (서버 빌드 시점에 자동으로 생성됨)
2. **Koyeb 가입 및 계정 연결**:
   - [Koyeb 공식 사이트](https://www.koyeb.com/)에 접속하여 가입 후 GitHub 계정을 연동합니다.
3. **App 생성**:
   - `Create Service` -> `GitHub`를 선택하고 방금 생성한 리포지토리를 연결합니다.
   - **Builder**: `Docker`를 선택합니다 (Dockerfile이 자동 탐색됩니다).
   - **Instance Type**: `Nano` (무료 티어)를 선택합니다.
   - **Ports**: 포트 번호 `8000`을 추가하고 프로토콜은 `HTTP`로 설정합니다.
4. **Deploy**:
   - `Deploy` 버튼을 누르면 약 2~3분 내에 빌드가 완료되고 고유의 무료 도메인 주소(예: `xxxx.koyeb.app`)가 생성됩니다. 참가자들에게 이 주소를 공유하세요!

## 4. 참가자들을 위한 머신러닝 세션 안내 룰

세션을 원활하게 운영하기 위해 참가자들에게 아래와 같이 배포하고 가이드하세요.

### 제공할 자료
1. `data/train.csv` (정답 `Churn` 레이블 포함)
2. `data/test.csv` (정답 `Churn` 레이블 제외)
3. 베이스라인 코드 주피터 노트북 (학습, 예측 및 결과 저장용 가이드)

### 제출 파일(.csv) 규격
참가자가 제출할 파일은 `test.csv` 파일의 모든 고객 아이디를 포함해야 하며, `Customer_ID`와 `Churn` 열로 이루어져 있어야 합니다. (행 수 30,000행 필수)

**제출 예시 (`submission.csv`):**
```csv
Customer_ID,Churn
C070001,0
C070002,1
C070003,0
...
```

## 5. 관리자(운영진) 꿀팁 ⚙️

- **제출 횟수 늘려주기**: `main.py`의 `MAX_DAILY_SUBMISSIONS = 5` 상수를 원하는 숫자로 바꾼 뒤 서버를 다시 가동(또는 푸시)하면 즉시 적용됩니다.
- **최종 순위 공개 (대회 종료 시)**:
  - 홈페이지 맨 아래의 `관리자 전용 설정` 버튼을 클릭합니다.
  - 관리자 비밀번호인 `dna_ml_admin_secret_2026`을 입력하고 `Private 리더보드 공개`를 클릭합니다.
  - 즉시 모든 팀의 70% 데이터에 대한 **최종 순위(Private F1-Score)**로 정렬이 변환되며, 우리 팀 제출 이력에서도 본인의 Private Score를 확인할 수 있게 됩니다.
